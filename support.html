Support

Project help and issue reporting.

For bugs, feature requests, and usage questions, use the issue tracker or contact details linked from the relevant Crunchy Codes project. When reporting a problem, include the project version, operating system, environment details, and clear reproduction steps when possible.

Security

If you need to report a security concern, do not post sensitive details in a public issue. Use the private security reporting channel listed in the project repository, or contact the maintainer directly.

Response times

Crunchy Codes projects are independently maintained. Support requests are reviewed as availability allows, with priority given to bugs, regressions, and security-related reports.

Project-specific details

Some tools, including future editor extensions, may have their own setup notes, support channels, or compatibility details. Prefer the instructions linked from that project when they are available.

Codex Local Meter

For Codex Local Meter bugs or feature requests, use the project's GitHub issue tracker. Include your extension version, VS Code version, operating system, and whether the diagnostics command can read your Codex folder.
